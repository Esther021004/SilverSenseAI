# 🎯 실행 순서 간단 정리

## ⚡ 실행해야 하는 파일만 정리

### 실행 파일 목록

| 파일명 | 실행 시기 | 설명 | API 키 필요? |
|--------|----------|------|-------------|
| `build_vectorstore.py` | **처음 1번만** | 벡터 DB 구축 | ❌ |
| `test_search_only.py` | 검색만 테스트 | 문서 검색 기능 테스트 | ❌ |
| `example_usage.py` | 지침 생성 테스트 | 응급 지침 생성 예제 | ✅ |
| `quick_start.py` | 선택사항 | 환경 확인 스크립트 | 선택 |

### 실행하지 않는 파일들

- `rag/` 폴더 안의 파일들 → **라이브러리** (import해서 사용)
- `README_RAG.md`, `시작하기.md` → **문서** (읽기만)
- `requirements.txt` → **설정 파일** (pip install에만 사용)

## 🚀 실제 실행 순서 (최소한)

### 최소 실행 (API 키 없이)

```bash
# 1단계: 패키지 설치 (한 번만)
pip install -r requirements.txt

# 2단계: 벡터 스토어 구축 (한 번만)
python build_vectorstore.py

# 3단계: 검색 기능 테스트
python test_search_only.py
```

**이것만 실행하면 끝!** (검색 기능만 사용)

---

### 전체 기능 사용 (API 키 필요)

```bash
# 1단계: 패키지 설치 (한 번만)
pip install -r requirements.txt

# 2단계: 벡터 스토어 구축 (한 번만)
python build_vectorstore.py

# 3단계: .env 파일에 API 키 추가
# OPENAI_API_KEY=sk-...

# 4단계: 지침 생성 예제 실행
python example_usage.py
```

## 📋 파일별 역할 설명

### ✅ 실행하는 파일들

1. **`build_vectorstore.py`**
   - 역할: 문서를 읽어서 벡터 DB 생성
   - 실행 횟수: 처음 1번만 (문서 추가할 때만 재실행)
   - 시간: 5-15분

2. **`test_search_only.py`**
   - 역할: 문서 검색 기능만 테스트
   - 실행 횟수: 언제든지
   - 시간: 몇 초

3. **`example_usage.py`**
   - 역할: 실제 응급 지침 생성 예제 (3가지 시나리오)
   - 실행 횟수: 언제든지
   - 시간: 몇 초 (API 호출 포함)

4. **`quick_start.py`** (선택)
   - 역할: 환경 확인 + 간단한 테스트
   - 실행 횟수: 처음 한 번 정도
   - 시간: 몇 초

### 📚 라이브러리 파일들 (실행 안 함)

`rag/` 폴더 안의 파일들은 **라이브러리 코드**입니다:
- 다른 파일에서 `from rag import ...` 형태로 import해서 사용
- 직접 실행하지 않습니다

## ❓ Q&A

**Q: 모든 .py 파일을 다 실행해야 하나요?**  
A: 아니요! `build_vectorstore.py` + `example_usage.py` (또는 `test_search_only.py`)만 실행하면 됩니다.

**Q: `rag/` 폴더 안의 파일은 언제 실행하나요?**  
A: 실행하지 않습니다. 다른 파일에서 import해서 사용하는 라이브러리입니다.

**Q: `build_vectorstore.py`를 매번 실행해야 하나요?**  
A: 아니요. 처음 1번만 실행하면 됩니다. 문서를 추가했을 때만 재실행하세요.

**Q: 가장 빠르게 시작하려면?**  
A: 
```bash
python build_vectorstore.py
python test_search_only.py
```
이 두 개만 실행하세요!
